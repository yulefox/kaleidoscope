# 测试数据库

一段时期内, 手头玩的东西大致差不多, 可以建一个私用的测试数据库. 把需要用到的一些配置, 数据都丢到测试库里. 不同的项目也可以直接使用.

耦合也不算重, 比较容易被替换.

也不用反复写一些测试数据.

## mongo

`Dial` 与集群建立会话, 如果是连接到同一集群, 通常只需调用一次, 后续通过 `New`/`Clone` 方法建立新会话.

```
[mongodb://][user:pass@]host1[:port1][,host2[:port2],...][/database][?options]
```

`Close` 释放资源.
